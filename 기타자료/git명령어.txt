Github에 계속 업데이트 하는법
---------------------------------------------------------------------
1. 추가할 파일 더하기
git add .

2. 히스토리 만들기
git commit -m "first commit"
또는 
git commit -m "second commit"

3. Github로 올리기
git push origin master(브런치이름)

Github로 팀프로젝트 하는법
---------------------------------------------------------------------
1. Github에서 소스코드 다운로드
 => 주소는 깃허브에서 들고와야함
폴더이름은 선택사항이다 (즉 없어도됨) 폴더이름을 줄경우에는 그 폴더가 새로 생성이 되면서 그 안에 코드들이 다운로드가 되고, 폴더이름을 안줄경우엔 깃허브 프로젝트 이름으로 폴더가 자동으로 생기고 그안에 코드들이 다운로드된다

git clone 주소 폴더이름

2. Github에서 내 브렌치(branch)만들기
git checkout -b 브렌치이름

3. 내 브렌치에 소스코드 업데이트하기
git add .
git commit -m "first commit"
git push origin 브렌치이름

4. 마스터 브렌치에 소스 가져오기(pull)
git pull origin master

5. 브렌치끼리 이동하는 법
git checkout 브렌치이름



------------------------------------------------------------------
오류 처리 방법
1. 커밋할 내용이 있는지 확인합니다
git status

2, 커밋을 수행합니다:
git commit -m "Initial commit"

3. 현재 브랜치 이름을 확인합니다
git branch

4. 확인된 브랜치로 푸시합니다
git push origin main
아니면  git push -u origin main


5. 마스터 브랜치 이름변경
git branch -m master main
# 또는 원하는 이름으로 변경
# git branch -m [새로운_브랜치_이름]

6. 원격 리포지토리의 기본 브랜치와 로컬 브랜치가 다른 경우:
원격 리포지토리에는 이미 main 브랜치가 있는데 로컬에는 master 브랜치만 있는 경우, 로컬에서 git pull을 먼저 수행하여 원격의 변경 사항(브랜치 정보 포함)을 가져온 후 다시 push를 시도해 볼 수 있습니다.

git pull origin [원격_브랜치_이름, 예: main]
# 병합 후
git push origin [원격_브랜치_이름, 예: main]
